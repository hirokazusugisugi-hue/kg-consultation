# AI診断システム 運用要件定義 — バックオフィスの活用と運営フロー

---

## 1. AI診断で何ができるようになるか

### 1-1. 現状のバックオフィス業務

| 業務 | 現状 | 担当 |
|------|------|------|
| 相談内容の把握 | Zoom録画/音声を再生して確認 | 管理者が手動 |
| 相談の要約作成 | なし（リーダーの記憶頼み） | — |
| 課題の整理 | リーダーが口頭 or レポートで報告 | リーダー |
| 品質管理 | アンケート（Q2-Q10）のみ | 相談者の主観評価 |
| 傾向分析 | なし | — |
| フォローアップ | リーダー裁量 | リーダー |
| レポート作成 | リーダーが手動作成（3日期限） | リーダー |

### 1-2. AI診断導入後に可能になること

| 業務 | AI導入後 | 担当 | 自動/手動 |
|------|---------|------|-----------|
| **相談要約** | 300字の要約を自動生成 | AI | 自動 |
| **課題抽出** | 企業の課題を構造化リストで自動抽出 | AI | 自動 |
| **テーマ・業種分類** | 相談内容から自動分類 | AI | 自動 |
| **品質評価** | 文字起こしから客観的にアドバイス品質を評価 | AI + 管理者確認 | 半自動 |
| **傾向分析** | 蓄積データから業種別・テーマ別の傾向を集計 | 管理者 | 手動（集計関数） |
| **フォローアップ提案** | 次回相談の推奨事項を自動提案 | AI | 自動 |
| **レポートドラフト** | 文字起こし+AI分析からドラフト自動生成 | AI + リーダー校正 | 半自動 |
| **コンサルタント評価** | 品質スコアの担当者別蓄積・比較 | 管理者 | 手動（スプレッドシート） |

## 2. スプレッドシート設計

### 2-1. 「AI診断」シート（新規作成）

AI分析結果を1件1行で蓄積するバックデータシート。

| 列 | 項目 | 入力元 | 説明 |
|----|------|--------|------|
| A | 申込ID | 自動 | 予約管理シートとの紐付けキー |
| B | 相談日 | 自動 | 確定日時（Q列）から転記 |
| C | 企業名 | 自動 | D列から転記 |
| D | 業種 | 自動 | H列から転記 |
| E | 相談テーマ | 自動 | I列から転記 |
| F | リーダー | 自動 | Y列から転記 |
| G | 参加メンバー | 自動 | 日程設定H列から転記 |
| **H** | **AI要約** | **AI自動** | 相談内容の300字要約 |
| **I** | **主要課題** | **AI自動** | カンマ区切り（最大5件） |
| **J** | **AI業種分類** | **AI自動** | AIが判定した詳細業種 |
| **K** | **AIテーマ分類** | **AI自動** | 経営戦略/財務/人事/マーケ/IT/法務/その他 |
| **L** | **緊急度** | **AI自動** | 高/中/低 |
| **M** | **AI品質スコア** | **AI自動** | 1-5（AIの初期評価） |
| **N** | **管理者品質スコア** | **管理者手動** | 1-5（管理者が確認・修正） |
| **O** | **フォローアップ提案** | **AI自動** | 次回向けの推奨事項 |
| **P** | **キーワード** | **AI自動** | カンマ区切り（最大10個） |
| Q | フォローアップ実施 | 管理者手動 | 済/未/不要 |
| R | 管理者メモ | 管理者手動 | 自由記述 |
| S | 分析日時 | 自動 | AI実行完了日時 |
| T | ステータス | 自動 | 完了/エラー |

### 2-2. 手動入力と自動入力の分担

```
【自動（AIが入力）】           【手動（管理者が入力）】
┌──────────────────┐        ┌──────────────────┐
│ H: AI要約         │        │ N: 管理者品質スコア │
│ I: 主要課題       │        │ Q: フォローアップ実施│
│ J: AI業種分類     │        │ R: 管理者メモ       │
│ K: AIテーマ分類   │        └──────────────────┘
│ L: 緊急度         │
│ M: AI品質スコア   │         管理者はAIの出力を確認し、
│ O: フォローアップ │         必要に応じてN列でスコアを
│ P: キーワード     │         修正、R列にコメントを記入
└──────────────────┘
```

**ポイント:** AIの出力は「参考値」として自動記入され、管理者が確認・修正する運用。
AI品質スコア（M列）と管理者品質スコア（N列）を分けることで、AIの評価精度を検証できる。

## 3. 運用フロー

### 3-1. 相談完了からAI診断までの自動フロー

```
相談終了
  │
  ├─ [Zoom相談の場合]
  │   ↓ 自動
  │   Zoom録画完了 → 録画URL自動取得（既存）
  │   ↓ 自動
  │   動画をさくらサーバーに保存
  │   ↓ 自動
  │   音声抽出（ffmpeg）
  │
  ├─ [対面相談の場合]
  │   ↓ スタッフ手動
  │   ポータルから音声アップロード（既存）
  │
  ↓ 自動
  文字起こし実行（Speech-to-Text API）
  ↓ 自動
  Google Docsに全文保存（既存）
  ↓ 自動（★新規）
  AI分析エンジンに送信
  ↓ 自動
  分析結果をスプレッドシート「AI診断」シートに書込
  ↓ 自動
  管理者にメール通知：「〇〇社のAI診断が完了しました」
  ↓
  【ここから管理者の作業】
```

### 3-2. 管理者の日常運用

#### 毎日（相談がある日）

| 時間帯 | 作業 | 所要時間 | 内容 |
|--------|------|----------|------|
| 相談後2-3時間 | AI診断結果の確認 | **3-5分/件** | メール通知を受けて「AI診断」シートを確認 |
| ↓ | 品質スコアの確認・修正 | ↑に含む | M列（AI評価）を見てN列（管理者評価）を入力 |
| ↓ | フォローアップ判断 | ↑に含む | O列（AI提案）を見てQ列（実施有無）を記入 |
| ↓ | メモ記入（必要時のみ） | 1-2分 | R列に補足コメント |

#### 具体的な作業手順

**Step 1: メール通知を確認**
```
件名: [AI診断完了] 株式会社〇〇 (2026/03/15)
本文:
  申込ID: KG-2026-0315-001
  企業名: 株式会社〇〇
  リーダー: 山田太郎
  AI要約: 製造業の海外展開に関する相談。主に東南アジア市場への
          進出における現地パートナーの選定と資金調達が課題...
  主要課題: 海外展開戦略, 現地パートナー選定, 資金調達
  緊急度: 中
  AI品質スコア: 4/5
  → スプレッドシートで詳細を確認: [リンク]
```

**Step 2: スプレッドシートで確認**
- 「AI診断」シートを開く
- 最新行のH列（AI要約）を読み、内容が妥当か確認
- M列（AI品質スコア）を確認

**Step 3: 管理者評価を入力**
- N列に管理者としてのスコア（1-5）を入力
  - AIと同意見なら同じ値を入力
  - 違和感があれば修正値を入力
- Q列にフォローアップの判断を入力
  - 「済」= 次回相談の案内を出す
  - 「未」= 後日判断
  - 「不要」= フォローアップ不要

**Step 4: （任意）メモ記入**
- R列に自由コメント
- 例:「リーダーの提案が的確。次回はIT活用の専門家も参加させたい」

#### 月次

| 作業 | 所要時間 | 内容 |
|------|----------|------|
| 月次レビュー | 15-30分 | AI診断シートの月間データを確認 |
| 傾向分析 | 10-15分 | 業種・テーマ・緊急度の分布を確認 |
| 品質レビュー | 10-15分 | コンサルタント別の品質スコア平均を確認 |

## 5. 品質管理の運用ルール

### 5-1. AI品質スコアの評価基準

AIが文字起こしテキストから自動評価する際の基準（プロンプトに組み込む）:

| スコア | 評価 | 基準 |
|--------|------|------|
| 5 | 優秀 | 課題を的確に把握し、具体的・実行可能なアドバイスを複数提示。相談者が行動に移せる内容 |
| 4 | 良好 | 課題把握は十分、アドバイスも概ね具体的。一部に深掘り余地あり |
| 3 | 標準 | 基本的な助言はできているが、具体性や独自性にやや欠ける |
| 2 | 要改善 | 課題把握が浅い、または一般的すぎるアドバイスに留まる |
| 1 | 不十分 | 相談者の課題に対して適切な助言ができていない |

### 5-2. 管理者による確認・修正の運用

```
AI品質スコア（M列）が入力される
  ↓
管理者が確認
  ├─ AIと同意見 → N列に同じスコアを入力
  ├─ AIより高い/低い → N列に修正スコアを入力
  └─ 判断つかない → N列を空欄のまま（後日確認）
```

**管理者スコアが不要なケース:**
- 時間がない場合はN列を空欄にしてもよい（AI評価のみで蓄積）
- 月次レビュー時にまとめて確認・入力も可

**AIと管理者の評価が大きく乖離する場合（2点以上の差）:**
- AI分析の精度に問題がある可能性 → プロンプト改善を検討
- または当該相談に特殊事情がある → R列にメモ

### 5-3. フォローアップの運用ルール

| 条件 | アクション |
|------|-----------|
| 緊急度「高」 | 1週間以内にフォローアップ連絡 |
| 緊急度「中」+ AI提案あり | 次回日程案内メールに提案内容を反映 |
| 緊急度「低」 | 通常運用（次回問い合わせ時に対応） |
| 品質スコア 2以下 | リーダーに個別フィードバック検討 |

## 6. データ蓄積と長期活用

### 6-1. 蓄積スケジュール

| 期間 | 想定件数 | データ量 | 活用レベル |
|------|---------|---------|-----------|
| 〜3ヶ月 | 15-30件 | 初期データ | AIスコアの精度検証、基準値の策定 |
| 3-6ヶ月 | 30-60件 | 中間データ | 傾向分析の開始、月次比較が可能に |
| 6-12ヶ月 | 60-120件 | 十分なデータ | 統計的な傾向把握、年次報告に活用 |
| 1年以降 | 120件〜 | 成熟データ | 前年比分析、長期トレンド把握 |

### 6-2. 初期（〜3ヶ月）の運用方針

導入直後はAIの出力精度を検証する期間。

```
1. 全件で管理者が N列（管理者スコア）を入力する
2. AI品質スコア（M列）と比較し、乖離が大きい件をメモ
3. 月次で乖離パターンを分析
4. 必要に応じてAIプロンプトを調整
5. 3ヶ月後にAIスコアの信頼度を評価 → 運用ルールを再設定
```

### 6-3. 安定期（3ヶ月〜）の運用方針

AIスコアの精度が確認できたら、管理者の負荷を軽減。

```
1. AI品質スコアが 3-5 の場合 → N列の入力は任意（AIを信頼）
2. AI品質スコアが 1-2 の場合 → 必ず管理者が確認・N列入力
3. 月次レビューで傾向分析を実施
4. 四半期ごとにAIプロンプトの見直し
```

## 7. 既存業務への影響

### 7-1. 管理者の作業時間の変化

| 業務 | 現状 | AI導入後 | 変化 |
|------|------|---------|------|
| 相談内容の把握 | 録画視聴 30-60分/件 | AI要約確認 2-3分/件 | **大幅削減** |
| 品質管理 | アンケートのみ確認 5分 | AI+アンケート確認 5-8分 | やや増加 |
| レポートドラフト | なし（リーダー依存） | AI下書き確認 5-10分 | 新規追加 |
| 傾向分析 | なし（手動不可能） | 月次集計シート確認 15-30分 | 新規追加 |
| **1件あたり合計** | **35-65分** | **12-20分** | **50-70%削減** |

### 7-2. リーダーへの影響

| 業務 | 現状 | AI導入後 | 変化 |
|------|------|---------|------|
| レポート作成 | 全文手動 60-120分 | AIドラフト確認・修正 20-40分 | **半減以上** |
| フォローアップ | 自己裁量 | AI提案を参考に実施 | 品質向上 |

### 7-3. 変更が不要なもの

以下の既存業務はAI診断導入後も**一切変更なし**:

- 予約受付フロー（LP → GAS → スプレッドシート）
- NDA同意フロー
- 日程設定・スコア計算
- Zoom自動作成
- アンケート自動送信
- ポータルの基本操作（ログイン、案件一覧、会場設定）
- メール通知の仕組み

## 8. 段階的導入プラン

### Phase A: 文字起こし + AI分析（最小構成）

```
既存の文字起こし完了
  ↓ 手動トリガー（ポータルのボタン or GASメニュー）
AI分析実行
  ↓
「AI診断」シートに結果書込
```

- **管理者の作業**: ボタンを押す + 結果確認
- **工数**: 小
- **コスト**: ¥160-175/件

### Phase B: 完全自動パイプライン

```
相談完了（ステータス変更）
  ↓ 自動
文字起こし → AI分析 → スプレッドシート書込 → メール通知
```

- **管理者の作業**: メール確認 + スコア入力のみ
- **工数**: 中
- **コスト**: 同上

### Phase C: レポートドラフト自動生成

```
AI分析完了
  ↓ 自動
レポートドラフト（Google Docs）生成
  ↓
リーダーに通知 → 確認・修正 → 承認 → 配信
```

- **リーダーの作業**: ドラフト修正のみ
- **工数**: 中〜大
- **コスト**: +¥15-30/件（AI追加呼出）
