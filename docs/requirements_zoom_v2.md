# Zoom連携 v2 要件定義 — 入室許可・録画保存先の改善

---

## 変更点サマリ（実装済み: Phase 1 + Phase 2）

| 項目 | v1（旧） | v2（実装済み） |
|------|-----------|-------------|
| 入室許可 | 管理者（ホスト）のみ入室許可可能 | **パスコード認証（待機室なし）、誰でもURL+パスコードで入室** |
| 待機室 | 常にON | **OFF（パスコードで認証）** |
| 録画方式 | Zoom クラウド録画（管理者アカウント） | **Zoomクラウド録画（変更なし）** |
| 録画内容 | 動画＋音声（Zoomデフォルト） | **動画＋音声（変更なし）** |
| 録画共有 | 手動 | **共有リンクを自動取得＋リーダーにメール通知** |
| 録画保存先 | Zoomクラウドのみ | **Zoomクラウド＋YouTube非公開自動アップロード（Cloud Function経由）** |
| YouTube連携 | なし | **非公開アップロード → 管理者がYouTube Studioでリーダーに共有** |

## 1. 入室許可の仕様変更

### 1-1. 現状の課題

- Zoom ミーティングは `waiting_room: true`（待機室ON）、`join_before_host: false`（ホストなしでは入室不可）
- ミーティングは管理者のZoomアカウント（`users/me`）で作成されるため、管理者がホスト
- 管理者が不在の場合、参加者は待機室から入室できない

### 1-2. 方式の選択肢

| 案 | 方式 | メリット | デメリット |
|----|------|---------|-----------|
| **A（推奨）** | リーダーを代替ホスト（Alternative Host）に設定 | セキュリティ維持、リーダーが入室管理可能 | リーダーにZoomアカウント（同組織 or ライセンスユーザー）が必要 |
| B | 待機室OFF＋パスコード認証 | 追加アカウント不要 | 誰でもリンクで入室可能（セキュリティ低下） |
| C | `join_before_host: true`＋待機室OFF | 管理者不在でも全員入室可 | セキュリティなし、入室管理不可 |

### 1-3. 推奨案A — 代替ホスト方式の仕様

#### 前提条件
- リーダーがZoomアカウントを持っていること（同組織ドメインまたはライセンスユーザー）
- メンバーマスタにリーダーの**Zoomメールアドレス**を追加管理

#### メンバーマスタ拡張

| 列 | 項目 | 説明 |
|----|------|------|
| M（新規） | Zoomメール | Zoomアカウントに紐づくメールアドレス |

> リーダーのGmailアドレスとZoomアカウントのメールが異なる場合を考慮

#### ミーティング作成時の変更

```
createZoomMeeting(data) の settings に追加:
  - alternative_hosts: "{リーダーのZoomメール}"
  - alternative_hosts_email_notification: true （リーダーに代替ホスト通知メール送信）
```

#### フロー

```
予約確定
  ↓
リーダー割り当て（既存）
  ↓
Zoom ミーティング作成
  ├─ ホスト: 管理者（既存）
  ├─ 代替ホスト: リーダー【新規】
  ├─ 待機室: ON（既存）
  └─ join_before_host: true【変更】（代替ホストが先に入室可能に）
  ↓
参加者に Zoom URL 送付
  ↓
当日: 管理者 or リーダーが入室許可を管理
```

#### 管理者不在時の動作

1. リーダーが代替ホストとしてミーティングを開始
2. リーダーが待機室の参加者を入室許可
3. 管理者が途中参加した場合 → 自動的にホスト権限を取得

## 2. 録画保存先の動的切り替え

### 2-1. 現状

- `auto_recording: 'cloud'` → 管理者のZoomクラウドに自動保存
- 録画ファイルの管理・共有は手動

### 2-2. 方式の選択肢

| 案 | 方式 | メリット | デメリット |
|----|------|---------|-----------|
| **A（推奨）** | 常にクラウド録画 → GASで自動ダウンロード＆Drive保存 | 完全自動化、保存先を柔軟に制御 | Zoom Webhook設定が必要、APIコール増 |
| B | 管理者参加→クラウド、不参加→ローカル録画 | シンプル | ローカル録画はリーダーのPC依存、手動アップロード必要 |
| C | 常にクラウド録画、手動でDriveに移動 | 実装不要 | 運用負荷が高い |

### 2-3. 推奨案A — 自動ダウンロード＆Drive保存方式

#### 概要

1. **すべてのミーティングをクラウド録画**（現行と同じ）
2. **録画完了後、GASトリガーで自動処理**:
   - Zoom APIで録画ファイルを取得
   - 保存先Driveフォルダを決定（管理者参加有無で分岐）
   - Google Driveにアップロード
   - スプレッドシートに録画URLを記録

#### 保存先ルール

| 条件 | 保存先 | 説明 |
|------|--------|------|
| 管理者が参加 | 管理者の指定Driveフォルダ | CONFIG で設定 |
| 管理者が不参加 | リーダー指定のDriveフォルダ | スプレッドシートまたはメンバーマスタで管理 |
| フォルダ未指定 | デフォルトフォルダ | CONFIG のフォールバック先 |

#### スプレッドシート拡張

##### 予約管理シート

| 列 | 項目 | 説明 |
|----|------|------|
| AB（新規） | 管理者参加 | TRUE/FALSE — 管理者がZoomに参加するか |
| AC（新規） | 録画URL | Drive上の録画ファイルURL |
| AD（新規） | 録画保存先 | DriveフォルダID（リーダー指定時） |

##### メンバーマスタ

| 列 | 項目 | 説明 |
|----|------|------|
| N（新規） | 録画保存先フォルダID | リーダーのデフォルト保存先（Drive共有フォルダ） |

#### CONFIG 拡張

```javascript
ZOOM: {
  ACCOUNT_ID: '',
  CLIENT_ID: '',
  CLIENT_SECRET: '',
  RECORDING: {
    DEFAULT_FOLDER_ID: '',        // デフォルト保存先DriveフォルダID
    ADMIN_FOLDER_ID: '',          // 管理者用保存先DriveフォルダID
    AUTO_DOWNLOAD: true,          // 自動ダウンロード有効/無効
    DELETE_CLOUD_AFTER_DOWNLOAD: false  // DL後にZoomクラウドから削除するか
  }
}
```

#### 録画取得フロー

```
ミーティング終了
  ↓
トリガー: 定期実行（1時間ごと）で未処理の録画をチェック
  ↓
Zoom API: GET /v2/users/me/recordings
  ↓
完了済みミーティングの録画を取得
  ├─ ミーティングIDからスプレッドシートの予約を特定
  ├─ 「管理者参加」列を確認
  │   ├─ TRUE → 管理者の指定フォルダにアップロード
  │   └─ FALSE → リーダー指定フォルダにアップロード
  ├─ 録画ファイル（MP4 + M4A）をDriveに保存
  ├─ 録画URLをスプレッドシートに記録
  └─ （オプション）Zoomクラウドから削除
```

### 2-4. 録画内容の仕様

| 項目 | 現行 | 変更後 |
|------|------|--------|
| 録画方式 | `auto_recording: 'cloud'` | `auto_recording: 'cloud'`（変更なし） |
| 動画（MP4） | 有効（Zoomデフォルト） | 有効 |
| 音声のみ（M4A） | 有効（Zoomデフォルト） | 有効 |
| 録画開始 | ミーティング開始時に自動 | ミーティング開始時に自動 |

> Zoom クラウド録画はデフォルトで**動画（MP4）＋音声（M4A）**の両方を保存します。
> 現在の `auto_recording: 'cloud'` 設定で既に対応済みです。

## 4. 技術的前提・制約

### Zoom API 制約

| 項目 | 制約 |
|------|------|
| 代替ホスト | 同じZoomアカウント（組織）のライセンスユーザーのみ指定可能 |
| 録画ダウンロード | 録画完了後にのみ取得可能（Webhookまたはポーリング） |
| 録画ファイルサイズ | 1時間のミーティングで約200MB〜500MB（画質による） |
| Server-to-Server OAuth | ミーティング作成・録画取得ともに対応済み |

### Google Drive 制約

| 項目 | 制約 |
|------|------|
| GASアップロード上限 | Blob 50MB制限 → 大容量ファイルは分割ダウンロードまたは Drive API v3 使用 |
| Drive容量 | 各ユーザーのDrive容量に依存 |
| 共有フォルダ | リーダー指定フォルダはGASサービスアカウントに編集権限が必要 |

### 対応が必要な事前作業

1. リーダーのZoomアカウント確認（メールアドレス把握）
2. メンバーマスタにZoomメール列を追加・データ入力
3. 録画保存用Driveフォルダの作成・共有設定
4. （案Aの場合）Zoomクラウド録画の容量確認

## 5. 段階的導入案

### Phase 1: 入室許可の改善（優先度高）
- メンバーマスタにZoomメール列追加
- `createZoomMeeting()` に代替ホスト設定を追加
- `join_before_host: true` に変更
- 動作検証

### Phase 2: 録画の自動Drive保存（優先度中）
- CONFIG・スプレッドシート列の拡張
- `processZoomRecordings()` 実装
- 定期トリガー設定
- 録画ダウンロード＆アップロードの動作検証

### Phase 3: 管理画面・通知（優先度低）
- 管理者参加フラグをLP or 管理画面から設定
- 録画完了通知（リーダー・管理者へ）
- マニュアル更新

## 6. 確認事項（ユーザーへの質問）

1. **入室許可方式**: 案A（代替ホスト）でよいですか？リーダー全員にZoomアカウントはありますか？
2. **録画保存先**:
   - 管理者参加時 → 管理者のZoomクラウドに残すだけでよいですか？それともDriveにも保存しますか？
   - 管理者不参加時 → リーダーごとに保存先フォルダを指定する運用で問題ないですか？
3. **Driveフォルダの構成**: 案件ごとに自動でサブフォルダを作成しますか？それとも1フォルダに全録画を保存しますか？
4. **録画の保持期間**: Zoomクラウド上の録画はDrive保存後に自動削除しますか？
5. **ファイルサイズ**: 90分のミーティング録画（約300-500MB）をGAS経由でDriveにアップロードする場合、50MB制限があるためDrive API v3の使用が必要です。この複雑さを許容しますか？
