1. システム概要

本システムは Google Apps Script（GAS） を基盤とした予約管理システムです。以下の4つのコンポーネントが連携して動作します。

  • LP（ランディングページ） — 相談者からの予約を受付するWebフォーム
  • Googleフォーム — メンバーの参加可能日を自動収集
  • スプレッドシート — 予約・日程・メンバーを一元管理
  • GAS — 上記すべてを連携し、メール通知・ステータス管理を自動実行
  • アンケートシステム — 相談後の満足度調査を自動配信・集計
  • オブザーバー専用ページ — 養成課程在学生が相談予定確認・NDA署名を行うポータル
毎月の日程生成、メンバーへのフォーム送信、スコア判定、予約受付処理はすべて自動化されています。管理者は主にスプレッドシート上のステータス管理と確認作業を行います。

3. 月次運用フロー（自動化済み）

以下のフローは毎月自動的に実行されます。管理者による手動操作は不要です。

毎月25日 自動実行
  • 翌々月の日程候補を自動生成
  • 全メンバーをデフォルト登録（全員参加可能として初期化）
  • オプトアウトフォームを自動作成
  • 全メンバーにメール送信（参加できない日をチェックするフォーム）
  • 管理者に通知メールを送信
翌月15日 自動実行
  • 確認フォームを自動作成
  • 全メンバーに2回目のメール送信（確認＋追加参加可能日の登録）
翌月21日 自動実行
  • スコア不足の枠を自動クローズ
  • 予約済み枠があれば警告メールを管理者に送信
  • 管理者に確定レポートを送信
以降（予約受付開始）
  • LPに予約可能日程が自動表示
  • 相談者が予約 → 自動処理（確認メール送信・スプレッドシート記録）
  • 管理者に通知メールが届く
すべてのフローはGASトリガーで自動実行されます。トリガーの設定状況は 10. トリガーの確認・管理 を参照してください。

5. お知らせ管理

  1. ウェブアプリURL + ?action=news-admin にアクセスします
  2. 「お知らせ追加」フォームから 日付 と 内容 を入力して追加します
  3. 一覧から 表示/非表示の切替 や 削除 が可能です
  4. LPのニュースバーに最新のお知らせが 自動表示 されます
お知らせの表示/非表示はスプレッドシートのC列（表示フラグ）でも直接制御できます。TRUE で表示、FALSE で非表示です。

6. 予約の管理

6.1 新規予約が入った場合

  • 管理者メールに 通知 が届きます
  • スプレッドシートの予約管理シートに 自動記録 されます
  • ステータスは 仮予約 として登録されます
  • 相談者には相談同意書確認リンク付きの 確認メール が自動送信されます

6.2 予約確定の流れ

オンライン相談（Zoom）の場合

  1. 相談者が相談同意書への同意を完了 → 自動でステータスが 確定 に更新されます
  2. 同意完了＋予約確定メールが相談者に自動送信されます
  3. Zoomミーティングが 自動作成 され、R列にURLが保存されます（パスコード付き、待機室なし）
  4. 日程設定シートH列の参加メンバー全員に確定通知メール・LINE通知が送信されます
  5. ミーティング終了後、録画の共有リンクがAB列に自動保存され、リーダーにメール通知されます
  6. 【YouTube連携有効時】録画がYouTubeに非公開で自動アップロードされ、AC列にURLが保存されます。管理者に「YouTube Studioでリーダーに共有してください」というメールが届きます

対面相談の場合

  1. 相談者が相談同意書への同意を完了 → ステータスが 同意済 に自動更新されます
  2. 相談者に同意完了確認メール（会場確保待ち案内）が自動送信されます
  3. 管理者に「会場確保依頼」の通知が届きます
  4. スタッフポータルの「案件」ページから「会場未設定」をタップし、会場を選択して確定します（N列に自動反映されます）
  5. ステータスを 確定 に変更 → 確定メールが相談者に自動送信されます
  6. 日程設定シートH列の参加メンバー全員に確定通知メール・LINE通知が送信されます
Zoomの場合は相談同意書への同意完了時に自動確定され、Zoomミーティングも自動作成されます。管理者による手動操作は不要です。自動作成に失敗した場合のみR列にURLを手動入力してください。
対面の場合は同意完了時に相談者へ会場確保待ち案内メールが自動送信されます。管理者は会場確保後、ステータスを「確定」に変更してください。
確定通知・リマインドの送信先: 日程設定シートのH列（参加メンバー）に登録されている全メンバーに送信されます。メンバーが未登録の場合のみ管理者メールアドレスにフォールバックします。

6.3 当日受付

  • 備考欄にコード 9999 が含まれている場合、当日受付 として処理されます
  • W列に TRUE が自動記録されます
当日受付コードは config.gs の WALK_IN_CODE で変更可能です。デフォルト値は 9999 です。

6.4 キャンセル自動検知

相談者からのキャンセルメールを自動検知し、以下の処理を実行します。

  • Gmail受信トレイを 10分おき にスキャン
  • 「キャンセル」「取消」「取り消し」「中止」を含むメールを検知
  • 該当予約のステータスを キャンセル に自動更新
  • 日程枠の解放 + Zoomミーティング削除（該当する場合）
  • 担当者・リーダー・参加予定メンバー全員にメール + LINE通知
  • 相談者にキャンセル受付確認メールを自動送信
  • 処理済みメールには「キャンセル処理済」ラベルを付与
スプレッドシート上でステータスを手動で「キャンセル」に変更した場合も、同様の処理（日程解放・Zoom削除・メンバー通知）が自動実行されます。

7. メンバー管理

メンバーの追加・編集はスプレッドシートの メンバーマスタシート で行います。

  • 新メンバー追加時は E列にメールアドレスを必ず入力 してください
  • 区分（D列）: 「診断士」または「オブザーバー」のいずれかを入力
  • 期（B列）: 1期〜4期
メールアドレスが空の場合、そのメンバーにはGoogleフォーム（参加可否アンケート）が送信されません。フォームが届かないという問合せがあった場合はまずE列を確認してください。
区分 期 スコア 説明
診断士 1期 / 2期 2点 経験豊富なメンバー。相談対応の主担当
オブザーバー 3期 / 4期 1点 研修中のメンバー。サポート役として参加

8. アンケートシステム

相談完了後に自動でアンケートメールを送信し、回答をスプレッドシートに記録するシステムです。

8.1 自動送信の仕組み

  • ステータスが 完了 に変更された案件が対象
  • 確定日時の 2時間後 にアンケートメールが自動送信されます
  • 1時間おきのトリガーでチェック・送信されます
  • 送信済みの案件には備考欄に「アンケート送信済」が記録されます
  • 相談者がアンケートに回答すると、管理者に通知されるほか、リーダーにもアンケート全結果（Q1〜Q11）がメール送信されます

8.2 アンケート項目

Q内容形式
Q1本プロジェクトを知ったきっかけ複数選択（SNS選択時はサブ選択）
Q2相談までの手続きはスムーズでしたか5段階評価
Q3相談の感想自由記述
Q4相談時間について5択（長すぎた～短すぎた）
Q5説明のわかりやすさ5段階評価
Q6課題解決の参考度5段階評価
Q7相談員の対応5段階評価
Q8具体的行動アドバイス5段階評価
Q9また受けたいか5段階評価＋理由
Q10他の方にすすめたいか5段階評価＋理由
Q11終了後レポート希望希望する/しない

8.3 セットアップ

  1. ウェブアプリURL + ?action=setup-survey にアクセスしてシートを初期化
  2. ウェブアプリURL + ?action=setup-survey-trigger にアクセスして自動送信トリガーを設定
アンケート送信のタイミング（相談終了後何時間後か）は config.gs の SURVEY.DELAY_HOURS で変更可能です（デフォルト: 2時間）。

9. オブザーバー専用ページ

養成課程在学生（オブザーバー）が秘密保持誓約書（NDA）への署名と相談予定確認を行うためのポータルページです。

「相談同意書」と「秘密保持誓約書（NDA）」の違い:
・相談同意書: 相談者（企業経営者）が予約時に同意する書類（nda.gs で処理）
・秘密保持誓約書（NDA）: オブザーバー（在学生）が各相談日ごとに署名する書類（observer.gs で処理）

9.1 アクセス方法

ウェブアプリURL + ?action=observer でアクセスできます。

9.2 機能一覧

  • 相談予定一覧: 確定済み・同意済の案件と、予約済みスロットを過去7日～未来分表示
  • NDA署名: 各相談予定に対し、タッチ対応の署名パッドでNDAに署名・PDF提出
  • 自動入力: 日付・担当者・企業名はスプレッドシートから自動取得

9.3 NDA提出フロー

  1. オブザーバーが専用ページにアクセスし、対象相談の「署名して提出」をクリック
  2. 氏名を入力し、署名欄にタッチ（またはマウス）で署名
  3. 「署名して提出」ボタンで署名済みPDFがGoogle Driveに保存
  4. オブザーバー本人に受領確認メールが自動送信される
  5. 管理者にメール通知が届く
  6. オブザーバーNDAシートに記録が追加される

9.4 セットアップ

ウェブアプリURL + ?action=setup-observer-nda にアクセスしてシートを初期化します。

署名済みNDAの保存先フォルダを指定する場合は config.gs の OBSERVER_NDA.DRIVE_FOLDER_ID にGoogle DriveのフォルダIDを設定してください。未設定時はDriveのルートフォルダに保存されます。

10. トリガーの確認・管理

GASエディタを開き、左メニューの「トリガー」（ 時計アイコン）をクリックして確認します。

以下の8つのトリガーが登録されていることを確認してください:

runFirstPolling 毎月25日 10:00
runReminderPolling 毎月15日 10:00
finalizeSchedule 毎月21日 10:00
sendSurveyEmails 1時間おき
checkCancellationEmails 10分おき
sendDailyReminders 毎日 9:00（担当者: 1週間前・3日前 / 相談者: 3日前・前日リマインド）
processZoomRecordings 1時間おき（Zoom録画チェック＋YouTube自動アップロード）
onSheetEdit スプレッドシート編集時（ステータス変更検知）
トリガーが消えた場合: GASエディタで setupAllTriggers() 関数を実行してください。これにより全トリガーが再設定されます。

11. GASコードの更新手順

  1. ローカル環境でコードを修正します
  2. ターミナルで以下のコマンドを実行します 
    clasp push --force
  3. GASエディタを開き、メニューから デプロイ → デプロイを管理 を選択します
  4. 鉛筆アイコン（）をクリックし、バージョンを 「新しいバージョン」 に変更します
  5. 「デプロイ」 をクリックして完了です
デプロイを更新しても URLは変わりません。LP側の変更は不要です。

トリガーが動かない トリガー未設定 GASエディタで setupAllTriggers() を実行してください
アンケートが送信されない ステータスが「完了」でない or トリガー未設定 ステータスを「完了」に変更し、sendSurveyEmailsトリガーが設定されているか確認
オブザーバーNDA提出でエラー Driveフォルダ権限不足 DRIVE_FOLDER_IDの権限を確認、またはIDを空にしてルートフォルダに保存
Zoomミーティングが自動作成されない Zoom API認証情報の未設定またはスコープ不足 GASエディタの「プロジェクトの設定」→「スクリプトプロパティ」で ZOOM_ACCOUNT_ID / ZOOM_CLIENT_ID / ZOOM_CLIENT_SECRET が設定されているか確認してください。また、Zoom MarketplaceでアプリのScopeに meeting:write:admin が含まれているか確認してください。
録画リンクがAB列に保存されない Zoom APIスコープ不足 Zoom MarketplaceでアプリのScopeに cloud_recording:read:list_user_recordings を追加してください。手動チェック: ?action=check-recordings で確認可能です。
キャンセルメールが検知されない トリガー未設定 or メールアドレス不一致 checkCancellationEmailsトリガーが設定されているか確認。相談者のメールアドレスが予約管理シートE列と一致しているか確認してください
Driveにファイルが保存されない Drive権限未承認 GASエディタで任意の関数を手動実行し、Drive権限の承認ダイアログを完了してください
YouTube URLがAC列に保存されない YouTube連携の未設定 1. CONFIG.YOUTUBE.ENABLED が true か確認
2. ?action=youtube-status でCloud Function URL/Secretが設定済みか確認
3. Cloud Functionのデプロイ状態とYouTube OAuth設定を確認してください
GASの実行ログは、GASエディタの左メニュー「実行数」（）から確認できます。エラーが発生している場合は赤い表示で記録されています。

14. リーダー自動選定

予約が 確定 になった時点でリーダーが「予定」として記録され、完了 になると「完了」に更新されます。リーダーが未設定の場合は完了時に参加メンバーの中から自動選定されます。

14.1 選定対象

  • 参加メンバー（H列）のうち、1期/2期の診断士のみが対象です
  • オブザーバー（3期/4期）はリーダー候補になりません

14.2 選定ロジック

以下のスコアを合算し、最も高いメンバーをリーダーに選定します。

評価項目スコア説明
業種マッチ+3メンバーマスタI列の得意業種と相談企業の業種が一致
テーママッチ+3メンバーマスタJ列の得意テーマと相談テーマが一致
キーワードマッチ+1相談内容に得意分野のキーワードが含まれる
履歴ペナルティ-2直近5回の相談でリーダーを担当していた場合
同点の場合はランダムで選定されます。選定結果は Y列 に自動記録され、リーダー履歴シート（4.7参照）にも詳細が保存されます。

14.3 リーダー履歴の流れ

確定時: 予定 完了時: 完了
  • ステータスが 確定 に変更された時点で、リーダー履歴シートに「予定」として記録されます
  • ステータスが 完了 に変更されると、既存の「予定」行が「完了」に自動更新されます
  • 手動でリーダーを設定した場合も、同様に「手動設定」として記録されます

14.4 管理者の操作

  • リーダー選定は自動で行われるため、通常は操作不要です
  • Y列を手動で変更することで、リーダーを上書きすることも可能です
  • メンバーマスタの I列（得意業種）・J列（得意テーマ） を適切に設定しておくと、選定精度が向上します
  • リーダー履歴シートで過去の選定結果やスコアを確認できます

15. レポート配信システム

相談後のアンケートでレポートを希望した相談者に対し、リーダーが作成した診断報告書を自動配信するシステムです。

15.1 フロー

  1. 相談完了後、アンケートが自動送信されます（2時間後）
  2. 相談者がアンケートQ11で「レポートを希望する」を選択
  3. リーダー（Y列）にレポート作成依頼メールが自動送信されます
  4. リーダーが専用アップロードページからPDF/Wordファイルを提出
  5. アップロード完了後、相談者に診断報告書が自動配信されます

15.2 期限管理

  • 提出期限: 相談日から 3日以内
  • ファイルサイズ上限: 5MB
  • 期限が近づくとリーダーにリマインドメールが自動送信されます
  • Z列で各案件のレポート状態（未依頼/依頼済/アップロード済/配信済/期限超過）を確認できます

15.3 レポート管理シート

レポート管理シートに、全案件のレポート進捗が自動記録されます。

A列申込ID B列相談日 C列相談企業 D列リーダー E列リーダーメール F列依頼日時 G列期限 H列アップロード日時 I列ファイルID J列ファイルURL K列配信日時 L列ステータス
レポート管理シートが存在しない場合は、ウェブアプリURL + ?action=setup-report にアクセスすると自動作成されます。リーダー履歴シートも同様に ?action=setup-leader-history で作成できます。

17. Zoom録画・YouTube自動アップロード

Zoomミーティング終了後、録画の共有リンクを自動取得し、さらにYouTubeに非公開動画として自動アップロードする機能です。

17.1 全体フロー

Zoomミーティング終了 自動
  • Zoomクラウド録画が自動保存される（auto_recording: 'cloud'）
1時間以内（processZoomRecordingsトリガー） 自動
  • Zoom APIから過去48時間の録画一覧を取得
  • スプレッドシートのZoom URL（R列）と照合し、未処理の録画を特定
  • 録画共有リンクをAB列に自動保存
  • リーダーに録画リンク＋パスワード付きメールを自動送信
YouTube非公開アップロード 自動
  • Cloud Function（Google Cloud）経由でZoom録画（MP4）をYouTubeに非公開アップロード
  • YouTube URLをAC列に自動保存
  • 管理者に「YouTube Studioでリーダーに共有してください」メールを送信
管理者の作業（手動）
  • メールに記載のYouTube Studio共有リンクをクリック
  • リーダーのメールアドレスを入力して共有
YouTubeの自動字幕（文字起こし）が利用可能なため、リーダーは診断報告書作成時に録画と字幕の両方を活用できます。

17.2 セットアップ・設定確認

操作 エンドポイント 説明
YouTube設定確認 ?action=youtube-status ENABLED、Cloud Function URL、Secretの設定状態を確認
YouTube設定登録 ?action=setup-youtube&cfUrl=URL&cfSecret=SECRET Cloud FunctionのURLとシークレットをスクリプトプロパティに保存
録画手動チェック ?action=check-recordings 録画チェック処理を即座に実行（トリガーを待たずに確認）
録画トリガー設定 ?action=setup-recording-trigger 1時間ごとの録画チェックトリガーをセットアップ

17.3 Cloud Function構成

項目 値
関数名 zoom-to-youtube
リージョン asia-northeast1（東京）
ランタイム Python 3.11（2nd gen）
GCPプロジェクト kg-consultation
URL https://asia-northeast1-kg-consultation.cloudfunctions.net/zoom-to-youtube
シークレット Secret Manager経由（SHARED_SECRET, YOUTUBE_CLIENT_ID, YOUTUBE_CLIENT_SECRET, YOUTUBE_REFRESH_TOKEN）
YouTube OAuthトークンの更新: YouTube APIのリフレッシュトークンは長期間有効ですが、Googleアカウントのパスワード変更やアプリの権限取消しにより無効化される場合があります。その場合はSecret Managerの youtube-refresh-token を更新してCloud Functionを再デプロイしてください。

リーダーに通知

ドラフト完了通知メール→レビュー・編集→最終承認

なし 処理待ち 処理中 完了 / エラー

この機能を利用するには config.gs で DIAGNOSIS.ENABLED = true を設定し、AI診断用Cloud Functionをデプロイする必要があります。

20.1 6軸評価

評価軸内部キースコア範囲

財務finance1-5
組織・人材organization1-5
マーケティングmarketing1-5
業務プロセスoperations1-5
IT・DXdigital1-5

総合スコアは0〜100点で算出されます。レーダーチャートで視覚的に確認可能です。

20.2 診断の実行
