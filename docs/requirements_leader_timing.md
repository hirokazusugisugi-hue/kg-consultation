# リーダー選定タイミング変更 要件定義（実装済み）

## 概要

リーダー選定のタイミングを「相談完了後」から「相談3日前まで」に前倒し済み。
3日前リマインドメール送信時点ではリーダーが確定している状態を実現。

**選定タイミングの方針:**

| 相談方法 | リーダー選定タイミング | 理由 |
|----------|----------------------|------|
| **対面** | 会場設定+確定時 | 確定自体が相談の数日前に行われるため、確定時点で十分 |
| **Zoom** | 3日前リマインド時 | NDA同意→即確定のため確定時は早すぎる。3日前にメンバーが確定してから選定 |

共通ルール: 3日前リマインド時点で未選定の場合はリマインド処理内で選定を実行する。

---

## 現状（As-Is）

| 項目 | 内容 |
|------|------|
| 選定タイミング | ステータスが「完了」に変更された時 |
| トリガー箇所 | `triggers.gs` の `handleStatusChange()` → `autoSelectLeaderOnComplete()` |
|               | `completion.gs` の `submitCompletionConfirm()` → `autoSelectLeaderOnComplete()` |
| 選定結果の記録 | 予約管理シート Y列（リーダー名）+ リーダー履歴シート（ステータス「完了」） |
| 問題点 | 相談当日までリーダーが未定 → 事前準備・役割分担ができない |

### 現状のステータスフロー

```
仮予約 → NDA同意済 → 書類受領 → 確定 ─── ･･･ ─── 完了
                                                    ↑ここでリーダー選定（遅い）
```

```

  → setVenueAndConfirm() → N列に会場名、O列を「確定」に変更
  → onSheetEdit → handleStatusChange()
  → autoSelectLeaderOnConfirm(rowIndex)  ← 新規追加
  → Y列にリーダー名を記録
  → 確定通知（スタッフ向け）にリーダー名を含める
```

### ケース2: Zoom相談（3日前リマインド時に選定）

```
（NDA同意時に自動確定 → この時点ではリーダー選定しない）

3日前リマインドトリガー（毎朝9時）
  → sendDailyReminders() 内で3日前の予約を検出
  → Y列が空ならリーダー選定を実行  ← 新規追加
  → autoSelectLeaderOnReminder(rowIndex)
  → Y列にリーダー名を記録
  → 3日前リマインドメール（スタッフ向け）にリーダー名を含める
```

### ケース3: 管理者がスプレッドシートで手動確定（対面）

```
管理者がO列を「確定」に変更
  → onSheetEdit → handleStatusChange()
  → autoSelectLeaderOnConfirm(rowIndex)  ← 新規追加
```

### ケース4: 3日前リマインド時の保証（対面・Zoom共通）

```
3日前リマインドトリガー
  → Y列が空の場合（何らかの理由で未選定）
  → autoSelectLeaderOnReminder(rowIndex)  ← フォールバック
  → リマインドメールにリーダー名を含める
```

## 変更対象ファイル

### 1. `gas/leader.gs`

**新規関数: `autoSelectLeaderOnConfirm(rowIndex)`**

対面相談の確定時に呼ばれる。

```
- 予約管理シートから行データを取得
- 既にリーダーが設定済み（Y列）の場合はスキップ（手動設定を尊重）
- 相談方法がZoomの場合はスキップ（Zoomは3日前リマインド時に選定）
- 日程設定シートから確定日の参加メンバーを取得
- selectLeader() でスコアリング → 最適なリーダーを選定
- Y列にリーダー名を記録
- リーダー履歴シートに「予定」として記録
```

**新規関数: `autoSelectLeaderOnReminder(rowIndex)`**

3日前リマインド時に呼ばれる。Zoom相談のメイン選定 + 対面のフォールバック。

```
- 予約管理シートから行データを取得
- 既にリーダーが設定済み（Y列）の場合はスキップ
- 日程設定シートから確定日の参加メンバーを取得
- selectLeader() でスコアリング → 最適なリーダーを選定
- Y列にリーダー名を記録
- リーダー履歴シートに「予定」として記録
- 選定結果を返す（リマインドメールに含めるため）
```

**既存関数: `autoSelectLeaderOnComplete(rowIndex)` の変更**

```
- 既にリーダーが設定済みの場合（確定時 or リマインド時に選定済み）:
  → リーダー履歴の「予定」を「完了」に更新（現状と同じ動作）
  → リーダーが当日の参加メンバーに含まれていない場合は再選定
- リーダー未設定の場合（何らかの理由で未選定）:
  → 従来通り選定を実行してフォールバック（安全策として残す）
```

### 2. `gas/triggers.gs`

**`handleStatusChange()` の確定処理に追加**

```javascript
// 確定に変更された場合（既存の確定処理の末尾に追加）
if (newStatus === STATUS.CONFIRMED) {
  // ... 既存の確定メール送信・Zoom作成・通知処理 ...

  // 対面相談の場合: リーダー自動選定（確定時に実行）
  // Zoom相談の場合: 3日前リマインド時に選定するためここではスキップ
  if (!data.leader) {
    try {
      autoSelectLeaderOnConfirm(rowIndex);
    } catch (leaderError) {
      console.error('リーダー選定エラー（確定時）:', leaderError);
    }
  }
}
```

**`sendDailyReminders()` の3日前リマインド処理に追加**

```javascript
// ── 相談者 + 担当者向け: 3日前リマインド ──
if (dateStr === threeDaysLaterStr) {
  // リーダー未選定の場合、ここで選定（Zoomのメイン選定 + 対面のフォールバック）
  if (!rowData.leader) {
    try {
      autoSelectLeaderOnReminder(i + 1);
      rowData = getRowData(i + 1);  // リーダー名を再取得
    } catch (leaderError) {
      console.error('リーダー選定エラー（3日前リマインド）:', leaderError);
    }
  }

  // 相談者向け（メール）
  sendReminderEmail3DaysBefore(rowData);

  // 担当者向け（リーダー名を含む）
  sendStaffReminderWithMembers_(rowData, '3日前');
}
```

## 考慮事項

### Zoomで確定時に選定しない理由

- Zoom相談はNDA同意→即確定のため、確定時点は相談日より大幅に早い場合がある
- 確定後にメンバー構成が変わる可能性が高い
- 3日前リマインド時点ならメンバーがほぼ確定しており、選定精度が高い
- リマインドメールに自然にリーダー名を含められる（追加メール不要）

### 対面で確定時に選定する理由

- 対面は会場確保後に確定するため、確定時点が相談日の数日前であることが多い
- 確定時にリーダーが決まれば、確定通知にリーダー名を含められる
- 3日前リマインド時にもフォールバックがあるため、万一の場合も安全

### 参加メンバーの確定度

- 確定時・リマインド時に選定したリーダーが当日欠席する可能性がある
- 対策: 完了時にリーダーが参加メンバーに含まれているか再チェック
  - 含まれていない場合: 再選定を実行
  - 含まれている場合: 履歴を「完了」に更新のみ

### 手動設定の尊重

- 管理者が事前にY列にリーダーを手動設定している場合、自動選定はスキップ
- 手動設定されたリーダーは履歴に「手動設定」として記録（現状と同じ）

## テスト観点

1. **対面相談・確定時**: ポータルで会場選択 → 確定 → Y列にリーダー名が入ること
2. **Zoom相談・確定時**: NDA同意 → 自動確定 → Y列は空のままであること（選定しない）
3. **Zoom相談・3日前**: 3日前リマインドトリガー実行 → Y列にリーダー名が入ること
4. **3日前フォールバック**: 対面で確定時に選定できなかった場合 → 3日前リマインドで選定されること
5. **手動リーダー設定済み**: Y列に手動でリーダー名がある場合 → 上書きされないこと
6. **リマインドメール内容**: スタッフ向け3日前リマインドにリーダー名が含まれること
7. **完了時**: リーダーの履歴が「予定」→「完了」に更新されること
8. **完了時・欠席フォールバック**: リーダーが当日参加メンバーにいない場合 → 再選定されること
9. **完了時・未選定フォールバック**: Y列が空のまま完了した場合 → 従来通り選定されること
