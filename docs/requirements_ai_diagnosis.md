# AI診断システム要件定義 — 録画保存・自動分析・スプレッドシート連携

---

## 概要

Zoom等のオンライン相談および対面相談の録画データを、さくらインターネットサーバーに保存・管理し、
AIによる自動診断（文字起こし＋内容分析）を実行、結果をGoogleスプレッドシートに集約するシステム。

### 現状の課題

| 項目 | 現状 | 課題 |
|------|------|------|
| 録画保存 | Zoomクラウド（容量制限あり） | 手動管理、容量逼迫リスク |
| 文字起こし | 手動 or ポータルから個別実行 | 1件ずつの手動操作が必要 |
| 相談内容分析 | なし（人力レビュー） | 時間・工数がかかる |
| バックデータ | なし | 傾向分析・品質改善の基盤がない |

### ゴール

```
録画データ → さくらサーバー保存 → 自動文字起こし → AI分析・診断 → スプレッドシート集約
```

## 2. Phase 1 — 録画データの自動保存

### 2-1. Zoom録画の自動取得

Zoomクラウド録画完了時に、録画ファイルをさくらサーバーに自動ダウンロード・保存する。

#### 方式A: Zoom Webhook → GAS → さくらPHP（推奨）

```
Zoom録画完了
  ↓ webhook（recording.completed）
GAS doPost() で受信
  ↓ ダウンロードURL取得
GAS → さくらPHP（download_zoom.php）にURL転送
  ↓
さくらPHP がZoomからファイルをダウンロード
  ↓
さくらSSD に保存: /files/video/{年月}/{申込ID}_{日付}.mp4
  ↓
GAS にURLをコールバック → スプレッドシート記録
```

#### 方式B: ポータルから手動アップロード

```
スタッフがポータルで [動画をアップロード] ボタンをクリック
  ↓
チャンクアップロード（既存の音声アップロード機能を拡張）
  ↓
さくらSSD に保存
  ↓
GAS にURLを保存
```

#### 方式C: 対面相談の録画デバイスから直接アップロード

```
対面相談終了後、スタッフが録画デバイスからファイルを選択
  ↓
ポータルからアップロード（方式Bと同じフロー）
```

### 2-2. ストレージ設計

#### ディレクトリ構成

```
/home/iba-consulting/www/site/api/audio/files/
  ├── audio/          ← 既存: 音声ファイル
  │   └── {年月}/
  ├── video/          ← 新規: 動画ファイル
  │   └── {年月}/
  │       └── {申込ID}_{日付}_{タイムスタンプ}.mp4
  └── extracted/      ← 新規: 動画から抽出した音声
      └── {年月}/
          └── {申込ID}_{日付}.m4a
```

#### 容量見積もり

| 解像度 | 60分/件 | 90分/件 | 月10件 | 年間120件 |
|--------|---------|---------|--------|-----------|
| 720p MP4 | ~200MB | ~300MB | ~2-3GB | ~24-36GB |
| 1080p MP4 | ~400MB | ~600MB | ~4-6GB | ~48-72GB |
| 音声のみ M4A | ~20MB | ~30MB | ~200-300MB | ~2.4-3.6GB |

**スタンダードプラン（300GB）の場合:**
- 720p: 約1,000〜1,500件まで保存可能（8〜12年分）
- 1080p: 約500〜750件（4〜6年分）

**推奨:** 720p保存 + 古いデータは年次で外部バックアップ（Google Drive等）

### 2-3. さくらサーバー側 新規ファイル

**`site/api/audio/download_zoom.php`** — Zoom録画ダウンロード受付API
- GASからPOSTリクエストを受信
- Zoom録画ダウンロードURLからファイル取得（cURL）
- ファイルサイズ上限: 1GB
- 保存先: `files/video/{年月}/`
- レスポンス: `{ success: true, url: "https://...", fileSize: ... }`

**`site/api/audio/extract_audio.php`** — 音声抽出トリガーAPI
- 動画ファイルパスを受け取り、ffmpegで音声抽出をバックグラウンド実行
- `exec("ffmpeg -i input.mp4 -vn -acodec aac -ab 128k output.m4a > /dev/null 2>&1 &")`
- 完了後GAS webhookにコールバック

### 2-4. 前提条件（さくらサーバー）

| 項目 | 要件 | 確認方法 |
|------|------|----------|
| ffmpeg | 音声抽出に必須 | `which ffmpeg` or 手動インストール |
| ストレージ空き | 最低50GB以上推奨 | `df -h` |
| PHP max_execution_time | 大容量ファイル処理のため600秒以上 | php.ini |
| cURL | Zoom APIからのダウンロードに必要 | `php -m | grep curl` |

## 3. Phase 2 — 自動文字起こしパイプライン

### 3-1. 処理フロー

```
動画保存完了（Phase 1）
  ↓ GAS webhook or 手動トリガー
さくらPython: extract_and_transcribe.py
  ├── 1. ffmpeg で音声抽出（動画→M4A）
  ├── 2. GCSに音声アップロード（既存処理）
  ├── 3. Speech-to-Text V2 batchRecognize（既存処理）
  ├── 4. 文字起こし結果をGAS webhookにPOST
  └── 5. GCS一時ファイル削除
  ↓
GAS:
  ├── Google Docsに文字起こし全文保存（既存処理）
  ├── スプレッドシート AD列: 文字起こし状態 → 完了
  └── スプレッドシート AE列: ドキュメントID記録
```

### 3-2. 既存機能との統合

現在の音声アップロード→文字起こしフローを拡張:

| 機能 | 既存 | 拡張 |
|------|------|------|
| 音声ファイル入力 | ポータルから手動アップロード | + 動画から自動抽出 |
| transcribe.py | 音声ファイルパスを引数で受取 | 変更なし（共通利用） |
| GASコールバック | 文字起こし結果POST | 変更なし（共通利用） |

## 4. Phase 3 — AI診断・分析

### 4-1. AI分析の対象と出力

文字起こしテキストをAI（LLM）に入力し、以下の診断結果を自動生成する。

#### 分析項目

| # | 項目 | 説明 | 出力形式 |
|---|------|------|----------|
| 1 | **相談要約** | 相談内容の300字以内の要約 | テキスト |
| 2 | **主要課題** | 企業が抱える課題（最大5つ） | リスト |
| 3 | **業種分類** | 相談企業の業種カテゴリ | 選択値 |
| 4 | **相談テーマ分類** | 経営戦略/財務/人事/マーケティング/IT/法務/その他 | 選択値（複数可） |
| 5 | **緊急度** | 高/中/低 | 選択値 |
| 6 | **アドバイス品質スコア** | コンサルタントの助言内容を1-5段階で評価 | 数値 |
| 7 | **フォローアップ提案** | 次回相談に向けた推奨事項 | テキスト |
| 8 | **キーワード** | 相談内容から抽出したキーワード（最大10個） | リスト |

#### プロンプト設計（概要）

```
あなたは中小企業診断士の上級アドバイザーです。
以下の経営相談の文字起こしテキストを分析し、所定のJSON形式で結果を出力してください。

【相談情報】
- 企業名: {company}
- 業種: {industry}
- 相談テーマ: {theme}
- 相談日: {date}

【文字起こしテキスト】
{transcript}

【出力形式】
{
  "summary": "...",
  "issues": ["...", "..."],
  "industry_category": "...",
  "theme_categories": ["...", "..."],
  "urgency": "高|中|低",
  "advice_quality_score": 1-5,
  "followup_suggestions": "...",
  "keywords": ["...", "..."]
}
```

### 4-2. AI分析エンジンの選択肢

| 案 | モデル | メリット | デメリット | 60分あたりコスト |
|----|--------|---------|-----------|-----------------|
| **A** | Claude API（Haiku 4.5） | 高精度、日本語強い、低コスト | API登録必要 | ~¥15-30 |
| **B** | Claude API（Sonnet 4.6） | 最高精度 | コスト高め | ~¥60-150 |
| **C** | Gemini API（Flash） | Google連携容易、無料枠大 | 精度はClaude比でやや劣る | ~¥5-15（無料枠あり） |
| **D** | Gemini API（Pro） | Google連携容易、高精度 | コスト中程度 | ~¥30-60 |

**コスト算出の前提:**
- 60分の文字起こしテキスト ≈ 10,000〜15,000字（約5,000〜8,000トークン）
- 入力 + プロンプト + 出力 ≈ 10,000〜15,000トークン/件

**推奨: 案A（Claude Haiku 4.5）** — コスト・精度のバランスが最良

### 4-3. 処理フロー

```
文字起こし完了（Phase 2）
  ↓ GAS webhookまたはトリガー
さくらPython: analyze.py
  ├── 1. 文字起こしテキストを取得（Google Docs or GAS経由）
  ├── 2. AI API呼出（プロンプト + テキスト）
  ├── 3. JSON結果をパース
  ├── 4. GAS webhookにPOST
  └── 5. エラー時リトライ（最大3回）
  ↓
GAS:
  ├── スプレッドシート「AI診断」シートに結果書込
  ├── 予約管理シートのステータス更新
  └── （オプション）管理者にメール通知
```

### 4-4. さくらサーバー側 新規ファイル

**`site/api/audio/analyze.py`** — AI分析ワーカー
- 引数: 文字起こしテキスト（ファイルパス or GAS URL）、行番号、webhook URL、APIトークン
- AI API呼出 → JSON結果取得
- GAS webhookにPOST
- バックグラウンド実行対応

**`site/api/audio/trigger_analyze.php`** — AI分析トリガーAPI
- GASから呼出
- analyze.pyをバックグラウンド実行

| H | テーマ分類 | テキスト | カンマ区切り（複数可） |
| I | 緊急度 | テキスト | 高/中/低 |

### 5-3. 集計・ダッシュボード用のデータ活用

蓄積データから以下の分析が可能になる:

| 分析項目 | 活用方法 |
|----------|----------|
| 業種別相談件数 | 業種分類の集計 → 需要傾向の把握 |
| テーマ別トレンド | テーマ分類の時系列分析 → 注力分野の特定 |

| 緊急度分布 | 緊急度の割合分析 → リソース配分の最適化 |
| キーワード頻出分析 | 経営課題のトレンド把握 |
| フォローアップ率 | 提案→次回相談への転換率 |

## 10. 検証ポイント

1. Zoom録画がさくらサーバーに自動保存される（webhook経由）
2. ポータルから手動で動画をアップロードできる
3. 動画から音声が自動抽出される（ffmpeg）
4. 抽出音声が自動で文字起こしされる（既存パイプライン）
5. 文字起こし完了後にAI分析が自動実行される
6. AI分析結果がスプレッドシート「AI診断」シートに書き込まれる
7. 予約管理シートのAI診断ステータスが更新される
8. ポータルからAI診断結果を閲覧できる
